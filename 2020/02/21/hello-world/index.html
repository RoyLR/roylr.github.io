Markdown 基本语法

发表于 2020-02-21 更新于 2020-02-24 分类于 语言 Valine：

Markdown是一种纯文本格式的标记语言。一提到语言，很多同学第一反应就是“难”、“复杂”等标签。

“这不应该是程序猿、攻城狮的事情吗？”

但经过我的实践，结论是一般正常人用5分钟时间学习，20分钟时间练习，即可掌握。在这里也向大家推荐此项语法用于写作。

Markdown的优势

一、书写过程流畅

一谈到文本编辑，相信很多人第一想到的就是Word。用Word进行文字编辑的时候，一般情况是用两个不连续的动作：输入文字的时候双手在键盘上，编辑文字则需视线和手离开输入框和键盘，去寻找和点击功能按钮。

这儿有两种是Word最有效率的编辑方式：

  • 一次性输入全部文字，再去编辑文字格式
  • 快捷键实现直接在文本内排版功能

而Markdown的书写流畅就体现在将这两个动作合成一个输入字符的动作。视线一直固定在光标处，手不需要脱离键盘，只是输入是使用不同的字符就能同时完成编辑和输入。这种体验类似纸笔时代的书写，使用者全部注意力都可以集中在大脑中的语句输出，而不用不停的切换。

二、格式不随编辑器而改变，导出分享方便

用于Word书写的工具有Office和WPS的各个不同版本，同时随着Mac用户的增多(特别是文字工作者使用Mac的比例越来越大)，Word的样式很容易出现问题。同时，Word的分享方式也非常受制约。
而Markdown则完全规避工具、版本带来的风险。用来分享，也远比富文本格式省心。

Markdown不适合使用场景

一、Markdown无法对段落进行灵活处理

在Word中你可以随意插入文本框，调整它的位置。尽管这并不是一个常见的用法，但是这意味着，Word能以段落为单位进行排版（Latex也能做到相似的效果），相比Markdown只能线性的对文字排版，专门的排版软件无疑是更能满足专业需求的。

二、Markdown对非文本排版能力差

Markdown引入图片很简单，但对于位置等排版方式很难做到自适应。
